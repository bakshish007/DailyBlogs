Git and Github:

Introduction to Git and Github

Git is a distributed version control system that helps track changes to files and coordinates work among multiple people on a project. Github is a web-based platform that provides hosting for software development version control using Git.

Git Basics

  • Git Clone: Clones a repository from Github to your local machine.
    git clone https://github.com/username/repository.git
  • Git Status: Checks the status of files in your repository.
    git status
  • Add and Commit: Stage changes and commit them with a message.
    git add filename
    git commit -m "Commit message"
  • Push Command: Push local changes to a remote repository.
    git push origin main

Branching and Merging

  • Git Branch: List, create, or delete branches.
    git branch
    git branch -M main
  • Merge Commands: Merge changes from one branch to another.
    git merge branchname
  • Pull Request: Propose changes and request that someone review and pull in your contribution.
    git pull origin branchname

Undoing Changes

  • Reset Staged Changes: Unstage files that were added.
    git reset filename
  • Reset Committed Changes: Undo the last commit.
    git reset HEAD~1
  • Reset Multiple Commits: Reset to a specific commit.
    git reset --hard commit-hash

Merging Conflicts

Merge conflicts occur when Git cannot automatically resolve differences between commits. They can be resolved manually using the 'git merge' command or through pull requests.

Forking a Repository

Forking creates a copy of a repository that you can modify and experiment with. You can contribute changes back to the original repository using pull requests.

Conclusion

Git and Github are powerful tools for version control and collaboration in software development. Understanding their basic commands and workflows is essential for effective project management and teamwork.
